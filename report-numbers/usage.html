Usage¶

Generic minter for report numbers with auto-increment per custom prefix.

Invenio-SequenceGenerator is a secondary component of Invenio, responsible for automatically generating report numbers in a safe manner (i.e. making sure that every time a client requests a new report number of a specific form, the server will provide him with a new unique report number).

Initialization¶

First create a Flask application:

>>> from flask import Flask
>>> app = Flask('myapp')

In order for the following examples to work, you need to work within an Flask application context so let’s push one:

>>> ctx = app.app_context()
>>> ctx.push()

Also, for the examples to work we need to create the database and tables (note, in this example we use an in-memory SQLite database):

>>> from invenio_db import db
>>> db.init_app(app)
>>> db.create_all()

Usage¶

First define your sequence (i.e. the specific form of your report numbers):

>>> from invenio_pidstore_reportnumber import new_sequence
>>> new_sequence('ID', 'File {counter}')

Then you can request new report numbers based on the sequence defined above. (NOTE: counters start from 1 and increment by one)

>>> from invenio_pidstore_reportnumber.models import get_next
>>> get_next('ID')
'File 1'
>>> get_next('ID')
'File 2'

User-defined keywords¶

Consider the case where you need to generate report numbers for files of different categories. Thus, your sequence definition should look like this:

>>> new_sequence('KW', '{category}: File {counter:03d}')

Note the formatting option of the counter.

Every time you request a new report number, you must also pass the category as a keyword argument.

>>> get_next('KW', category='PHOTOS')
'PHOTOS: File 001'
>>> get_next('KW', category='PHOTOS')
'PHOTOS: File 002'
>>> get_next('KW', category='VIDEOS')
'VIDEOS: File 001'

Hierarchical report numbers¶

It is possible to have nested sequences (i.e. sequences depending on another). This is achieved by placing a placeholder with the name of the parent sequence and making sure a valid report number of the parent is passed on each get_next call.

Consider the example of playlists of audio files for each year. We need two sequence definitionsThe definition, one for playlists and one for files. (NOTE: year is a user-defined keyword)

>>> new_sequence('PL', '{year}: Playlist {counter}')
>>> new_sequence('FL', '{PL} > Audio File {counter:02d}')

Let’s get some playlists for different years.

>>> get_next('PL', year=2015)
'2015: Playlist 1'
>>> get_next('PL', year=2015)
'2015: Playlist 2'
>>> get_next('PL', year=2016)
'2016: Playlist 1'

Now let’s get some files inside the playlists, generated above.

>>> get_next('FL', PL='2015: Playlist 1')
'2015: Playlist 1 > Audio File 01'
>>> get_next('FL', PL='2015: Playlist 1')
'2015: Playlist 1 > Audio File 02'
>>> get_next('FL', PL='2015: Playlist 2')
'2015: Playlist 2 > Audio File 01'
>>> get_next('FL', PL='2016: Playlist 1')
'2016: Playlist 1 > Audio File 01'
>>> get_next('FL', PL='2016: Playlist 1')
'2016: Playlist 1 > Audio File 02'

PID store integration¶

When the system detects that the Invenio-PIDStore module is installed, some additional validation is enabled. Specifically, when a parent’s report number is supplied, the system checks the PID store to ensure that it exists.

From the previous playlist example, if we would make this request to the server without Invenio-PIDStore installed in our system, we would get a new report number, otherwise an appropriate exception would be raised.

try:
    get_distribution('invenio-pidstore')

    get_next('FL', PL='2016: Playlist 2')
    # >>> E: Parent PID does not exist

except DistributionNotFound:
    get_next('FL', PL='2016: Playlist 2')
    # >>> '2016: Playlist 2 > Audio File 01'

Conventions¶

Conventions regarding template strings of sequences:

  • They must contain exactly one placeholder for the counter, named ‘counter’, which possibly contains conversion and/or format options.
  • They can contain an optional placeholder for a parent sequence, named as the parent sequence, defined when the new_sequence API function was called.
  • They can contain an arbitrary number of user-defined placeholders, which must be always passed as keyword arguments to the next API function.
